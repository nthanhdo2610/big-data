There are 3 6-sided dice. The first two are fair dice. They give each value 1/6 of the time. The third dice is not fair, 1/2 the time it shows the the value 1, and 1/10 of the time it will show any other side (sides 2, 3, 4, 5, 6). Pick a dice at random. Roll it 2 times. What is the probability of getting 1 in both rolls?



probability getting dice:<br/>P(1st) = 1/3<br/>P(2nd) = 1/3<br/>P(3rd) = 1/3<br/><br/>probability of getting 1 in both rolls for each chosen dice:<br/>P(1|1st) = 1/3 * (1/6*1/6) = 1/108<br/>P(1|2nd) = 1/3 * (1/6*1/6) = 1/108<br/>P(1|3rd) = 1/3 * (1/2*1/2) = 1/12<br/><br/>=> The probability of getting 1 in both rolls when picking a dice at random:<br/><br/>1/108 + 1/108 + 1/12 = 0.101851852<br/>

Compress the following (doc_id, frequency) list:
[(10, 10), (12, 15), (27, 128]
Using the following compression schemes:
a- Byte-aligned
b- Word-Aligned
c- Bit-aligned
d- Determine the final number of bytes in each compression case.

    

d-gaps
[(10, 10), (2, 15), (15, 128)]

sequence of non-zero integers
10, 10, 2, 15, 15, 128

convert to binary, then calculate n #bits and (2n-1) #bits
10 -> 1010 -> 4 -> 7
10 -> 1010 -> 4 -> 7
2 -> 10 -> 2 -> 3
15 -> 1111 -> 4 -> 7
15 -> 1111 -> 4 -> 7
128 -> 10000000 -> 8 -> 15

a- Byte-aligned
divide the binary into 7 bits, then combine the highest bit of 1 and count the bytes
10 -> 1010 -> 0001010 -> 10001010 -> 1 byte
10 -> 1010 -> 0001010 -> 10001010 -> 1 byte
2 -> 10 -> 0000010 -> 10000010 -> 1 byte
15 -> 1111 -> 0001111 -> 10001111 -> 1 byte
15 -> 1111 -> 0001111 -> 10001111 -> 1 byte
128 -> 10000000 -> 000000001|10000000 -> 2 bytes
=> Total: 7 bytes

b- Word-Aligned
sequence of the number of calculated n #bit
4,4,2,4,4,8

Applying the Simple-9 then, we get the result:
[0100] -> 5 -> [4,4,2,4,4]
[1000] -> 28 -> [8]
=> 2 words
=> Total: 4*2 = 8 bytes

c- Bit-aligned
Total of (2n-1) bit = 7+7+3+7+7+15 = 46 bits
=> Total: 46/8 = 6 bytes
